# mobile / Screen_Stock

## Purpose
Field-technician stock/inventory screen for a specific building or paired building-group ("Torre"). Technicians view current article quantities on hand, search/filter the list, receive new stock (increment existing or create new stock rows), correct a quantity, and switch which building/tower's stock they are viewing. One named user ("Perche, Adriana") is hardcoded to read-only (cannot add stock, cannot edit rows) — a crude non-role-based permission gate.

## Layout
Single canvas (640x1136 phone frame, drawn as background SVG rounded-rect). Header zone (y0-160): back-chevron IconButton (top-left), "swap tower" IconButton and dark "+" add-stock IconButton (top-right, both 67x67), building label "Edificio: {TorreStock}" (y139), search Input (y155). Body: a single vertical Gallery (y219-1090, TemplateSize=91) of stock rows — each row = white card, quantity Badge (blue pill, left), article name label (center), edit-pencil IconButton (right, hidden for the gated user). Empty state illustration (icon+heading+body text) shown instead when CollectStock has zero qty>0 rows. Three mutually-exclusive full-screen modal groups (each: 12%-black overlay + a bottom-anchored white rounded card ~55-65% of screen height, header text, divider line, one or two field placeholders, a gray Cancel button and a dark Accept/Save button) toggled by boolean context vars: PopUpAddStock, PopUpEditarStock, PopUpCambioTorreST. A full-screen loading overlay (embedded raster spinner image) toggles on `Loading`.

## components
- TextInput1 — search Input, placeholder "Buscar...", filters gallery by TextInput1.Text in ConcatArt_ST
- bt_addStock_SS — dark square IconButton ("+"); refreshes article catalog into CollectArticulos and opens the Add-Stock modal; DisplayMode.Disabled for NombreUser="Perche, Adriana"
- bt_backHome_SS — back-chevron IconButton; resets search, clears TorreStock/IDTorreStock, Navigate('Home Tecnico')
- GalleryStock — vertical list bound to CollectStock (Filter qty>0 and search match, sorted by ID then ConcatArt_ST); row children: Img_Fondo_SS (card bg, tap = Select(Parent), inert), bt_cantidad_SS (disabled blue quantity-badge button showing Cantidad_ST), lbl_art_SS (article name), Img_Edit_SS (pencil IconButton, opens Edit-Stock modal with RegistroArtEditStock=ThisItem; Visible=false for the gated user)
- bt_CambioTorre_SS — icon button that loads active buildings into CollectEdificios and opens the Change-Tower modal
- txt_Edificio_SS — label showing current building/tower name
- Img_NoStock_SS — empty-state illustration, visible only when no stock rows with qty>0 exist
- img_loading_SS — full-screen loading overlay (raster spinner), visible on global Loading flag
- Add-Stock modal (GroupAddStock_SS): article Combobox (cmbox_articuloAddSt_SS, options = catalog minus articles already stocked in this tower), quantity Input (input_Cant_ST), Cancel and Accept buttons (Accept disabled until article selected and qty>0)
- Edit-Stock modal (GroupEditStock_SS): quantity Input (input_EditCant_ST, defaulted to current qty), Cancel and Save/Accept buttons (Accept disabled until qty is a positive number), plus a small X close icon
- Change-Tower modal (GroupCambioTorre_SS): building Combobox (cmbox_ConfigTorre_SS bound to CollectEdificios), Cancel and Accept buttons (Accept disabled until a building is selected); Accept applies a hardcoded building-pairing rule before reloading stock

## modals
- Add-Stock (GroupAddStock_SS / PopUpAddStock): bottom-sheet with article Combobox (deduplicated against articles already stocked >0 in the current tower) + quantity Input + Cancel + Accept; Accept increments an existing '08.Stock' row or inserts a new one, and always inserts an '08.MovimientoStock' audit row (TipoMovimiento_MS "Nuevo" or "Editado")
- Edit-Stock (GroupEditStock_SS / PopUpEditarStock): bottom-sheet with a single quantity Input pre-filled from the tapped row's Cantidad_ST + Cancel + Save; Save overwrites '08.Stock'.Cantidad_ST and inserts a '08.MovimientoStock' audit row (TipoMovimiento_MS "Editado")
- Change-Tower (GroupCambioTorre_SS / PopUpCambioTorreST): bottom-sheet with a building Combobox (from '99.ABM_Edificios') + Cancel + Accept; Accept maps the chosen building through a hardcoded pairing rule (Admin/Admin2 share pool "16;17"; Hub/Nuñez share "1;8"; Palermo Hollywood/Dorrego share "4;7"; anything else uses its own ID) and reloads CollectStock filtered to that pool — encodes a real business rule that some physically distinct buildings share one stock pool
- Full-screen loading overlay (img_loading_SS, global `Loading` flag, no Group): a raster spinner image covering the whole canvas during any write/refresh sequence — not a popup with buttons, just a blocking indicator

## data_reads
- '99.ABM_Articulos' — Refresh() then Filter(Status_AR="Activo") into CollectArticulos, used by the Add-Stock article combobox (further filtered client-side to exclude articles whose Concat_AR already appears among CollectStock rows with qty>0 for the current tower)
- '99.ABM_Edificios' — Filter(Status_E="Activo") into CollectEdificios, populates the Change-Tower combobox (field shown: Edificio_E)
- '08.Stock' — Refresh() then FirstN(SortByColumns(ID,Descending),2000) filtered by (Text(IDTorreStock) in/= IDEdificio_ST) And Status_ST="Activo", re-seeding CollectStock after every write and after a tower change; the gallery itself reads only the in-memory CollectStock (Filter Cantidad_ST>0 And search-text in ConcatArt_ST, sorted by ID then ConcatArt_ST)
- CollectStock — in-memory collection that is NOT initialized on this screen (no OnVisible/App-level seed visible here); it is presumably populated by the prior screen ('Home Tecnico') when the user picks a building, then only re-populated here after writes/tower-switch

## data_writes
- Patch('08.Stock', <existing row>, {Cantidad_ST: oldQty + input_Cant_ST.Text}) — Add-Stock flow when the selected article already has a stock row for the current tower (increments quantity)
- Patch('08.Stock', Defaults('08.Stock'), {Título:"sumar", IDUnivoco_ST, Usuario_ST:NombreUser, Fecha_ST, Hora_ST, Codigo_ST, Articulo_ST, ConcatArt_ST, Cantidad_ST:input, Edificio_ST:TorreStock, IDEdificio_ST:Text(IDTorreStock), CondicionCorte_ST, Desde_ST:"Mobile", IDArticulo_ST, Status_ST:"Activo", MesAño_ST, VersionApp_ST}) — Add-Stock flow when no row exists yet for that article+tower (creates new stock row)
- Patch('08.Stock', RegistroArtEditStock, {Cantidad_ST: input_EditCant_ST.Text}) — Edit-Stock flow, overwrites quantity on the selected row
- Patch('08.MovimientoStock', Defaults('08.MovimientoStock'), {Título:"sumar", Usuario_MS:NombreUser, IDArticulo_MS, Articulo_MS, ConcatArticulo_MS, CantAnterior_MS, CantPosterior_MS, Cantidad_MS, IDEdificio_MS:IDTorreStock, Edificio_MS:TorreStock, Desde_MS:"Mobile - Stock", TipoMovimiento_MS:"Nuevo"|"Editado", Fecha_MS, Hora_MS, Status_MS:"Null", MesAño_MS, VersionApp_MS}) — audit/movement log row inserted on EVERY Add or Edit path (both branches of Add, and the Edit save)

## navigation
- bt_backHome_SS.OnSelect → Navigate('Home Tecnico', ScreenTransition.Fade) after resetting search text and clearing TorreStock/IDTorreStock context vars
- No other Navigate() calls on this screen — everything else (Add Stock, Edit Stock, Change Tower) is a local UpdateContext-driven modal toggle (PopUpAddStock / PopUpEditarStock / PopUpCambioTorreST booleans), not a screen navigation

## statuses
- Status_AR = "Activo" (article filter)
- Status_E = "Activo" (building filter)
- Status_ST = "Activo" (stock row filter/assignment)
- Status_MS = "Null" (literal string on every movement row — looks like an unused/legacy field, not a real state machine)
- TipoMovimiento_MS: "Nuevo" | "Editado" (movement type)
- Título (Title) field hardcoded to "sumar" on both '08.Stock' and '08.MovimientoStock' patches — generic/leftover default, not a meaningful business status
- Desde_ST = "Mobile" ; Desde_MS = "Mobile - Stock" (source-app tags)

## role_logic
Only one, non-scalable gate exists on this screen: a literal string comparison `NombreUser = "Perche, Adriana"` (not a profile/role lookup, no CollectPermisos or 99.ABM_ListaPermisosPerfilesV3 reference at all). It (a) disables bt_addStock_SS (DisplayMode.Disabled) and (b) hides Img_Edit_SS (the per-row edit-pencil icon), for that specific named user only — everyone else has full add/edit rights. When porting to Supabase, this must become a real permission/capability check (e.g. a boolean or role field on the user's profile), not a hardcoded name.

## react_mapping
Mobile-first single-page layout using Sumar UI Kit's mobile/table-to-cards conventions. Header: a mobile TopBar composition — leading IconButton (back), title Text ("Edificio: {tower}"), trailing IconButton pair (swap-tower, add-stock) — add-stock IconButton disabled via a permission flag (NOT a literal-name check; replace the "Perche, Adriana" hardcode with a real `canEditStock`/role capability from the new Supabase profile model). Search: kit Input (search variant) driving client-side filter. List: table-to-cards pattern — each CollectStock row becomes a Card with a StatusBadge/Badge (quantity, blue pill) + Text (article name) + trailing IconButton (edit pencil, hidden per the same permission flag); no Table component needed since there's only two data fields per row and this is mobile. Empty state: Card/EmptyState composition (icon + heading + body copy) in place of Img_NoStock_SS. The three PopUp* groups are NOT centered Modals in the original (cards sit in the bottom 55-65% of the screen) — map each to the kit's bottom Drawer/bottom-sheet recipe rather than the desktop right-Drawer or centered Modal: (1) Add-Stock sheet = Combobox (article, pre-filtered against already-stocked items) + numeric Input (quantity) + Cancel/Primary Button footer; (2) Edit-Stock sheet = numeric Input (pre-filled qty) + Cancel/Primary Button footer; (3) Change-Tower sheet = Select/Combobox (building) + Cancel/Primary Button footer, with the Admin/Admin2 & Hub/Nuñez & Palermo-Hollywood/Dorrego pairing rule reimplemented as a lookup table/config rather than an inline Switch. Loading overlay → kit's blocking Spinner/loading-overlay component instead of a raster GIF. Quantity inputs are plain numeric Input, not MoneyInput (these are unit counts, not currency; the one price-edit code path is commented out/dead in the source and should not be ported). Use Toast for save-success/error feedback on the two Patch flows (add/edit), which the Power Apps version has no equivalent of today.

# Data Model

## Tables
- edificios <- 99.ABM_Edificios. PK id bigint (preserve SP ID). Columns: nombre text NOT NULL UNIQUE (<- Edificio_E), pais text (<- Pais_E), status estado_activo NOT NULL DEFAULT 'activo' (<- Status_E 'Activo'/'Inactivo'), created_at timestamptz (<- Created), updated_at timestamptz (<- Modified). Title is unused boilerplate ('sumar') -> drop.
- articulos <- 99.ABM_Articulos. PK id bigint (preserve SP ID). Columns: codigo text (<- Codigo_AR), nombre text NOT NULL (<- Articulo_AR), precio_unitario numeric(12,2) (<- PrecioUnitario_AR, stored as string), corte text (<- Corte_AR, meaning unclear - see open questions), status estado_activo (<- Status_AR), detalle text (<- Detalle_AR), created_at/updated_at. Concat_AR ('codigo - nombre') is derived -> drop, expose as generated column or view if the UI needs it.
- unidades <- 99.ABM_TipoUnidades. PK id bigint (preserve SP ID - CRITICAL: 07.OrdenesTrabajo.IDActivo_OT and 19.Ventilaciones.IDHabitacion_VE point at this SP ID). Columns: id_client text (<- Title, display 'ID_Client'), depto text (<- field_1 Depto_ABMUnid), torre text (<- field_2 Torre_ABMUnid), edificio_id bigint NULL FK->edificios ON DELETE RESTRICT (resolve torre name = edificios.nombre at load; torre names are compared against Edificio_Usr in formulas - confirm), tipo_depto text (<- field_3 TipoDepto_ABMUnid), tipo_sector text (<- TipoSector_ABMUnid), status estado_alta (<- Status_ABMUnid 'Alta'), frecuencia_ventilacion_dias numeric (<- Frecuencia_ABMUnid), requiere_ventilacion boolean (<- Ventilacion_ABMUnid 'SI'/'NO'), created_at/updated_at.
- frecuencias <- 99.ABM_Frecuencias. PK id bigint. Columns: nombre text (<- Title), dias numeric NOT NULL (<- field_1 Frecuencia_FE), status estado_activo (<- field_2 Status_FE). Pure picklist; ventilaciones stores the dias VALUE, not an FK (desktop Screen_Ventilaciones: Frecuencia_VE:cmbox_frecuencia_CVE.Selected.Value).
- usuarios <- 00.Usuarios. PK id bigint (preserve SP ID - 19.Ventilaciones.IDAsignado_VE = RegistroUser.ID points here). Columns: auth_user_id uuid NULL UNIQUE FK->auth.users(id) ON DELETE SET NULL, nombre text (<- field_1 Nombre_Usr), apellido text (<- field_2 Apellido_Usr), concat_name text (<- field_3 ConcatName_Usr - KEEP: it is the text-join key used by Tecnico_IN/Asignador_OT/Usuario_* everywhere), usuario_app text UNIQUE NOT NULL (<- UsuarioApp_Usr, the login), dni numeric (<- field_4 DNI_Usr), fecha_nac date (<- field_5 FechaNac_Usr dd/mm/yyyy string), mail text (<- field_6 Mail_Usr), num_cel text (<- field_7 NumCelNomenclado_Usr), edificio_id bigint NULL FK->edificios ON DELETE SET NULL (resolve Edificio_Usr name -> edificios.nombre; evidence: Filter('08.Stock',Edificio_ST = VarUser.Edificio_Usr)), edificio_default text (<- EdificioDefault_Usr), pais text (<- Pais_Usr; Pais_U duplicate -> pick non-null), perfil perfil_usuario NOT NULL (<- field_9 Perfil_Usr), validado boolean (<- field_10 Validado_Usr), wapp_default text (<- field_13), mnt_global text (<- MNTGlobal_Usr), aplicacion text (<- Aplicacion_U, multi-app tag e.g. 'Desktop'), es_testing boolean (<- UsuarioTesting_Usr), status estado_usuario (<- Status_Usr 'ALTA'/'BAJA'), legacy_id_usr numeric (<- field_11 ID_Usr), created_at/updated_at. Password_Usr (field_8) is NOT migrated as a column - see auth_notes.
- perfiles_permisos <- 99.ABM_ListaPermisosPerfilesV3. PK id bigint. Columns: modulo text NOT NULL (<- field_1 Modulo_LPP: 'Home','Stock','Compras','Aprobaciones','Ordenes de Trabajo','Ventilaciones','ABM' per Switch(ThisItem.Modulo_LPP) navigation), admin text (<- field_2), operador text (<- field_3), tecnico text (<- field_5), recepcion text (<- field_4), compras text (<- Compras_LPP), jefe_operativo text (<- JefeOperatico_LPP), orden numeric (<- field_8 SortByColumns_LPP), imagen_path text (<- field_9 Imagen_LPP, base64 -> Storage), imagen_no_selected_path text (<- field_10), aplicacion aplicacion_app NOT NULL (<- field_11 Aplicacion_LPP 'Desktop'/'Mantenimiento'), status estado_activo (<- field_12 Status_LPP). Recommended v2: normalize to permisos(perfil perfil_usuario, modulo, aplicacion, permitido boolean) once per-role cell values are inspected.
- iconos_app <- 99.ABM_IconosApp. PK id bigint. Columns: icono_path text (<- Icono_IA, base64 image -> upload to Storage bucket 'branding', store path), app text (<- App_IA 'Desktop'/'Mobile'), descripcion text (<- Descripcion_IA, e.g. 'Logotipo','Zocalo Inferior'). Lookup key in App.OnStart is (App_IA, Descripcion_IA) -> UNIQUE(app, descripcion).
- emails_notificacion <- 99.ABM_Emails. PK id bigint. Columns: modulo text NOT NULL (<- Modulo_E: 'OT','Compra','Aprobaciones' per LookUp(CollectMails,Modulo_E=...)), emails text (<- EmailConcat_E, semicolon-joined recipient list - keep as text or text[]), edificio text (<- Torre_E, display 'Edificio_E'), a_tecnolav text (<- ATecnolav_E), status estado_activo (<- Status_E).
- ordenes_trabajo <- 07.OrdenesTrabajo. PK id bigint (preserve SP ID - IDRevision_IN and aprobacion-style lookups use it). Columns: id_univoco text UNIQUE NOT NULL (<- IDUnivoco_OT, display IDUnivoco_IN; app-generated '(OT)-XXX-ddmmyyyyhhmmss'; retained because bitacoras/fotos/repuestos/documentos join on it), status estado_ot NOT NULL (<- Status_OT), tipo tipo_ot (<- Tipo_IN 'ORDEN DE TRABAJO'/'SOLICITUD OT'), prioridad prioridad (<- Prioridad_IN), tipo_trabajo text (<- TipoTrabajo_IN), tipo_tarea text (<- TipoTarea_IN), tipo_prioridad text (<- RequiereParada_OT display TipoPrioridad_OT), unidad_id bigint NULL FK->unidades ON DELETE RESTRICT (<- IDActivo_OT), torre text (<- Torre_OT, denormalized torre/edificio name), departamento text (<- Sector_OT display Departamento_OT, denorm from unidades.depto), concat_activo text (<- ConcatActivo_IN, derived 'torre - depto' -> optional drop), tecnico_id bigint NULL FK->usuarios ON DELETE SET NULL (resolve Tecnico_IN = usuarios.concat_name), asignador_id bigint NULL FK->usuarios ON DELETE SET NULL (resolve Asignador_OT = concat_name), user_carga_id bigint NULL FK->usuarios (resolve UserCarga_IN = usuario_app), fecha_inicio date (<- FechaInicio_OT dd/mm/yyyy), fecha_cierre date (<- FechaCierre_OT), fecha_asignada date (<- FechaAsignada_IN), dias_estimado numeric (<- DiasEstimado_IN), personas_requeridas numeric (<- PersonasRequeridas_IN), detalle text (<- Detalle_IN), observaciones text (<- Observaciones_IN), obs_resuelto text (<- ObservacionesResuelto_IN), obs_asignacion text (<- ObservacionesAsignacion_IN), obs_cierre text (<- ObservacionesCierre_IN), orden_revision_id bigint NULL FK->ordenes_trabajo(id) ON DELETE SET NULL (<- IDRevision_IN, self-FK: replicated OT stores source OT's SP ID), problema text (<- Problema_OT), desde text (<- Desde_IN 'Desktop'/'Mobile'), version_app text (<- Version_IN), hora time (<- Hora_IN), created_at timestamptz (<- Created). Drop FechaMesAno_IN/FechaMes_IN (derived from dates).
- bitacoras <- 15.Bitacoras. PK id bigint. Columns: orden_trabajo_id bigint NOT NULL FK->ordenes_trabajo ON DELETE CASCADE (<- IDOrden_BC, holds OT.IDUnivoco_IN text; resolve via id_univoco map), id_univoco_bitacora text UNIQUE (<- IDUnivocoBitacora_BC, app-generated timestamp key; retained: fotos_bitacora joins on it), descripcion text (<- DescripcionBitacora_BC), fecha timestamptz (<- FechaBitacora_BC + Hora_BC), usuario_id bigint NULL FK->usuarios (resolve User_BC), version_app text (<- Version_BC).
- fotos_bitacora <- 16.FotoBitacora. PK id bigint. Columns: orden_trabajo_id bigint NULL FK->ordenes_trabajo ON DELETE CASCADE (<- IDUnivocoOT_FB = OT.IDUnivoco_IN), bitacora_id bigint NULL FK->bitacoras ON DELETE CASCADE (<- IDBitacora_FB = bitacoras.IDUnivocoBitacora_BC), foto_path text (<- Foto_FB: today a base64/long string column; migrate binary to Storage bucket 'bitacoras' and store the object path), created_at.
- repuestos_ot <- 10.RepuestosOT. PK id bigint. Columns: orden_trabajo_id bigint NOT NULL FK->ordenes_trabajo ON DELETE CASCADE (<- IDUnivoco_ROT = OT.IDUnivoco_IN), articulo_id bigint NULL FK->articulos ON DELETE RESTRICT (<- IDArticulo_ROT, numeric; matches 08.Stock.IDArticulo_ST which is articulos SP ID), repuesto text (<- Repuesto_ROT, denorm ConcatArt), cantidad numeric (<- Cantidad_ROT string), edificio text (<- Edificio_ROT, torre name text), usuario_id bigint NULL FK->usuarios (resolve Usuario_ROT = concat_name), fecha timestamptz (<- Fecha_ROT + Hora_ROT), status estado_activo (<- Status_ROT), version_app text (<- VersionApp_ROT). Drop MesAño_ROT.
- stock <- 08.Stock. PK id bigint (preserve SP ID - mobile Patch('08.Stock',LookUp(...,ID=RowID)) uses it). Columns: id_univoco text (<- IDUnivoco_ST '(STK)-...'), articulo_id bigint NOT NULL FK->articulos ON DELETE RESTRICT (<- IDArticulo_ST number = articulos SP ID; evidence: IDArticulo_ST = cmbox_articulo_AS.Selected.ID over CollectArticulosABM='99.ABM_Articulos'), cantidad numeric NOT NULL DEFAULT 0 (<- Cantidad_ST string), precio_unitario numeric(12,2) (<- PrecioUnitario_ST), condicion_corte numeric (<- CondicionCorte_ST, copied from articulos.Corte_AR = min-stock threshold), status estado_activo (<- Status_ST), usuario_id bigint NULL FK->usuarios (resolve Usuario_ST), fecha timestamptz (<- Fecha_ST + Hora_ST), ultima_mod text (<- UltimaMod_ST), desde text (<- Desde_ST), version_app text (<- VersionApp_ST). Codigo_ST/Articulo_ST/ConcatArt_ST are denorm copies of articulos -> drop. Edificio linkage goes through stock_edificios (below) because IDEdificio_ST is multi-valued.
- stock_edificios (NEW junction). PK (stock_id, edificio_id). stock_id bigint FK->stock ON DELETE CASCADE, edificio_id bigint FK->edificios ON DELETE RESTRICT. Populated at migration by splitting 08.Stock.IDEdificio_ST on ';' (values like '16;17','1;8','4;7' are hard-coded merged-building pairs in Screen_Compras receive logic; Edificio_ST holds 'Admin ; Admin 2' etc.). Single-building rows produce one junction row.
- movimientos_stock <- 08.MovimientoStock. PK id bigint. Columns: articulo_id bigint NULL FK->articulos ON DELETE SET NULL (<- IDArticulo_MS - DIRTY: some Patches store Codigo_AR text instead of the ID; resolve numeric->articulos.id, else match articulos.codigo, else NULL + keep raw), articulo_raw text (<- IDArticulo_MS original value, audit), concat_articulo text (<- ConcatArticulo_MS), articulo text (<- Articulo_MS), cant_anterior numeric (<- CantAnterior_MS), cant_posterior numeric (<- CantPosterior_MS), costo_anterior numeric (<- CostoAnterior_MS), costo_posterior numeric (<- CostoPosterior_MS), stock_min_anterior numeric (<- StockMinAnterior_MS), stock_min_posterior numeric (<- StockMinPosterior_MS), edificio_id bigint NULL FK->edificios ON DELETE SET NULL (<- IDEdificio_MS; also multi-valued '16;17' - take first or NULL + keep raw), edificio_raw text (<- IDEdificio_MS original), edificio text (<- Edificio_MS), edificio_traslado text (<- EdificioTraslado_MS), desde text (<- Desde_MS e.g. 'Desktop - Recibir Compra','Mobile - OT'), tipo_movimiento text (<- TipoMovimiento_MS: seen 'Nuevo','Existente','Asignacion Repuesto' - open set, keep text), cantidad numeric (<- Cantidad_MS), usuario_id bigint NULL FK->usuarios (resolve Usuario_MS = concat_name), fecha timestamptz (<- Fecha_MS + Hora_MS), version_app text. Status_MS is always the literal 'Null' -> drop. This is an append-only audit table: FKs SET NULL, never CASCADE.
- salidas_stock <- 09.SalidaStock. PK id bigint (preserve SP ID - edited via LookUp(ID = IDSalidaEdit)). Columns: articulo_id bigint NULL FK->articulos ON DELETE RESTRICT (<- IDArticulo_SS number), concat_articulo text (<- ConcatArt_SS), tecnico_id bigint NULL FK->usuarios ON DELETE SET NULL (resolve Tecnico_SS = concat_name; combobox Items = Distinct(Filter(CollectUsers,Perfil_Usr='Tecnico'),ConcatName_Usr)), tipo tipo_salida_stock (<- Tipo_SS), fecha_salida date (<- FechaSalida_SS), fecha_reingreso date (<- FechaReingreso_SS), uso text (<- Uso_SS), centro_de_costo text (<- CentroDeCosto_SS - building/torre name text; optionally resolve to edificio_id), cantidad numeric (<- Cantidad_SS), usuario_id bigint NULL FK->usuarios (resolve User_SS = usuario_app), fecha timestamptz (<- Fecha_SS + Hora_SS), version_app text (<- Version_SS). Drop FechaMes_SS/FechaAno_SS.
- compras <- 14.Compras. PK id bigint (preserve SP ID - aprobaciones.IDCompra_AP stores it; flows use Text(InfoRecibir.ID) as folder name). Columns: id_compra text UNIQUE (<- IDCompra_C '(BUY)-XXX-...'; detalle_compras + aprobaciones join on it), usuario_id bigint NULL FK->usuarios (resolve UsuarioCompra_C display 'Usuario_C' = concat_name of creator), usuario_compra text (<- UsuarioCompra_C0 display 'UsuarioCompra_C' - requester chosen in combobox; resolve to usuarios if it matches concat_name, else keep text), urgencia text (<- UrgenciaPedido_C), observacion text (<- Observacion_C), obs_recibir text (<- ObsRecibir_C), fecha date (<- Fecha_C), cantidad_total numeric (<- CantidadTotal_C), monto_total numeric(14,2) (<- MontoTotal_C), status estado_compra NOT NULL (<- Status_C), cargo text (<- Cargo_C, only read once in UI - legacy), sector_pedido text (<- SectorPedido_C, never written in screens - legacy), version_app text, created_at (<- Created). Drop Hora_C into created_at, drop MesAño_C.
- detalle_compras <- 15.DetalleCompras. PK id bigint. Columns: compra_id bigint NOT NULL FK->compras ON DELETE CASCADE (<- IDCompra_DC = compras.IDCompra_C business key; resolve via map), id_univoco text (<- IDUnivoco_DC '(DTC)-...'), articulo_id bigint NULL FK->articulos ON DELETE RESTRICT (<- IDArticulo_ST internal name, display IDArticulo_DC, number), articulo text (<- Articulo_ST display Articulo_DC, denorm), edificio_id bigint NULL FK->edificios ON DELETE RESTRICT (<- IDEdificio_DC number; combobox_EdificioCompra_SC.Items = CollectEdificios from '99.ABM_Edificios'), edificio text (<- Edificio_DC denorm), cantidad numeric (<- Cantidad_ST display Cantidad_DC), recibido numeric (<- Recibido_DC), costo_unitario numeric(12,2) (<- PrecioUnitario_ST display CostoUnitario_DC), cant_min numeric (<- CantMin_DC), costo_total numeric(14,2) (<- CondicionCorte_ST display CostoTotal_DC - NOTE SP internal names here are recycled from a copy of 08.Stock; MAP BY DISPLAY NAME), status estado_activo (<- Status_ST display Status_DC 'Activo'/'Inactivo' = soft-delete of line), usuario_id bigint NULL FK->usuarios (resolve Usuario_DC), fecha timestamptz (<- Fecha_DC + Hora_DC), version_app text.
- aprobaciones <- 16.Aprobaciones. PK id bigint. Columns: compra_id bigint NOT NULL FK->compras ON DELETE CASCADE (three redundant SP refs all point to the same compra: IDCompra_AP = compras SP ID, IDCompra_A = compras SP ID, IDUnivoco_AP = compras.IDCompra_C business key - collapse to ONE FK, resolve by SP ID first, fallback business key), status estado_aprobacion NOT NULL (<- Status_AP), fecha date (<- Fecha_AP), urgencia text (<- Urgencia_AP), tecnico text (<- Tecnico_AP - copied from compra requester name; resolve to usuarios where possible), obs_compra text (<- Obscompra_AP), fecha_aprobada date (<- FechaAprobada_AP), obs_rechazo text (<- ObsRechazo_AP), user_gen_id bigint NULL FK->usuarios (resolve UserGen_AP = usuario_app), user_aprob_id bigint NULL FK->usuarios (resolve User_AP = usuario_app), cantidad numeric (<- Cantidad_AP), monto numeric(14,2) (<- Monto_AP), sector text (<- Sector_AP), version_app text (<- Version_AP), hora time (<- Hora_AP). Drop the four FechaMesAño*/FechaAño* derived columns.
- ventilaciones <- 19.Ventilaciones. PK id bigint (preserve SP ID - 'TopRentals-FotoVentilacion'.Run(RegistroVentilacion.ID,...) keys photos by it). Columns: estado estado_ventilacion NOT NULL (<- field_1 Estado_VE), unidad_id bigint NULL FK->unidades ON DELETE RESTRICT (<- field_4 IDHabitacion_VE; evidence LookUp('99.ABM_TipoUnidades',ID = RegistroVentilacion.IDHabitacion_VE)), direccion_edificio text (<- field_2 DireccionEdificio_VE), edificio text (<- field_3 Edificio_VE denorm; matched against usuarios.Edificio_Usr), habitacion text (<- field_5 Habitacion_VE denorm), frecuencia_dias numeric (<- field_6 Frecuencia_VE - copied VALUE from unidades.frecuencia_ventilacion_dias, NOT an FK to frecuencias), fecha_ultima date (<- field_7 FechaUltima_VE), proxima_limpieza date (<- field_8 ProximaLimpieza_VE), fecha_programada date (<- field_11 FechaProgramada_VE), obs_adelanto text (<- field_12), obs_resuelto text (<- field_13), asignado_id bigint NULL FK->usuarios ON DELETE SET NULL (<- field_15 IDAsignado_VE = usuarios SP ID; evidence IDAsignado_VE = RegistroUser.ID; field_14 Asignado_VE is the denorm name), fecha_asignado timestamptz (<- field_16 + field_17 Hora), version_asignado text (<- field_18), fecha_finalizacion timestamptz (<- field_19 + field_22 Hora), version_resuelto text (<- field_23), es_incidente boolean (<- field_24 EsIncidente_VE 'SI'/'NO'), orden numeric (<- field_25 Orden_VE). Drop field_9/10/20/21 (derived month/year strings).
- documentos <- Documentos library (metadata only; binaries go to Storage). PK id bigint. Columns: nombre text (<- file name), storage_path text NOT NULL (bucket object key mirroring 'Ordenes/{id_univoco}/...', 'Compras/{compra_id}/...', 'Bitacoras/{id_univoco}/...'), carpeta text ('Ordenes'|'Compras'|'Bitacoras' from folder path), orden_trabajo_id bigint NULL FK->ordenes_trabajo ON DELETE CASCADE (<- IDOrdenes column = OT.IDUnivoco_IN, evidence Filter(Documentos, IDOrdenes = ThisItem.IDUnivoco_IN); for Ordenes/Bitacoras folders also derivable from folder name), compra_id bigint NULL FK->compras ON DELETE CASCADE (from folder 'Compras/{SP ID}'), content_type text, created_at timestamptz (<- Created).

## Auth
Today login is 100% custom and INSECURE: Screen_Login (desktop) and Login (mobile) do a plain-text comparison against the 00.Usuarios list - Filter(CollectUsers, UsuarioApp_Usr = input_user.Text And Password_Usr = input_password.Text) - where Password_Usr (field_8) is a plain string column readable by every app user. There is no hashing, no lockout, and the whole user list (passwords included) is downloaded to the client via ClearCollect(CollectUsers, Filter('00.Usuarios', Status_Usr = 'ALTA')). Roles: Perfil_Usr single-role string per user (Admin, Tecnico, Gerencia, Compras, Supervisor Ventilaciones, Recepcion, Operador). Per-app authorization: 99.ABM_ListaPermisosPerfilesV3 rows filtered by Aplicacion_LPP = 'Desktop' (desktop app) or 'Mantenimiento' (mobile app) + Status_LPP = 'Activo'; each row is a module (Modulo_LPP) with one column per role; the mobile app additionally blocks Tecnico from desktop login (Perfil_Usr='Tecnico' is rejected on Screen_Login desktop). The logged user record is kept in VarUser/RegistroUser and its ConcatName_Usr / UsuarioApp_Usr are stamped into every transactional row (that is why those two columns must stay in public.usuarios as resolution keys). Supabase mapping: (1) create auth.users per active usuario - email = Mail_Usr when present and valid, otherwise a synthetic alias like {usuario_app}@users.toprentals.internal since login is by username, and store usuario_app in auth user_metadata so the client can log in via signInWithPassword after a username->email lookup RPC; (2) DO NOT carry the plain-text passwords forward - generate random passwords and force a reset flow (admin.createUser with email_confirm true + password reset links), or if the client insists on continuity, seed the existing passwords once via admin API and force change on first login; (3) public.usuarios keeps the profile (perfil, edificio_id, status) with auth_user_id uuid referencing auth.users - the app reads the profile after sign-in; (4) enforce roles with RLS policies keyed on a JWT custom claim or a lookup of usuarios.perfil via auth.uid(), replacing the client-side CollectPermisos gating; keep perfiles_permisos as the menu/module configuration table.

## Migration
Node/TS script, wipe-and-load. SharePoint side: Microsoft Graph API with client-credentials (Sites.Read.All application permission). Env vars: MS_TENANT_ID, MS_CLIENT_ID, MS_CLIENT_SECRET, SP_SITE_URL (resolve site id via GET /sites/{hostname}:/{server-relative-path}), and per-list DISPLAY names (SP_LIST_USUARIOS='00.Usuarios', SP_LIST_OT='07.OrdenesTrabajo', ... 21 vars or one JSON map) resolved to list ids via GET /sites/{siteId}/lists?$filter=displayName eq '{name}' (never hardcode GUIDs). Read items with GET /sites/{siteId}/lists/{listId}/items?$expand=fields&$top=2000 following @odata.nextLink; address columns by their INTERNAL names from the schema dumps (field_1, MesA_x00f1_o_C, etc.) - beware 15.DetalleCompras where internal names are recycled from 08.Stock (IDArticulo_ST is displayed as IDArticulo_DC): the mapping tables above are keyed by internal name for exactly this reason. Documentos is a document library: enumerate via /sites/{siteId}/drives -> /root/children recursively, download each driveItem, upload to Supabase Storage. Supabase side: SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY, inserts via supabase-js (service role bypasses RLS) in batches of ~500. Wipe: single 'TRUNCATE documentos, fotos_bitacora, bitacoras, repuestos_ot, aprobaciones, detalle_compras, compras, salidas_stock, movimientos_stock, stock_edificios, stock, ventilaciones, ordenes_trabajo, perfiles_permisos, emails_notificacion, iconos_app, usuarios, unidades, frecuencias, articulos, edificios RESTART IDENTITY CASCADE' via an exec_sql RPC or direct pg connection. Insert order (respects FKs): 1) edificios, articulos, frecuencias; 2) unidades (resolve torre->edificio by name), usuarios (create auth.users first via admin API, then profile rows); 3) perfiles_permisos, iconos_app, emails_notificacion; 4) ordenes_trabajo WITHOUT orden_revision_id, then a second UPDATE pass to set it (self-FK; or declare it DEFERRABLE INITIALLY DEFERRED); 5) stock + stock_edificios (split IDEdificio_ST on ';'); 6) movimientos_stock, salidas_stock; 7) compras; 8) detalle_compras, aprobaciones; 9) bitacoras; 10) fotos_bitacora, repuestos_ot; 11) ventilaciones; 12) Documentos library -> Storage + documentos rows. PK strategy: bigint identity, INSERT with the original SharePoint ID as explicit id, then SELECT setval(pg_get_serial_sequence(...), max(id)) per table - this makes every numeric text-join (IDActivo_OT, IDAsignado_VE, IDArticulo_*, IDEdificio_*, IDCompra_AP, IDRevision_IN) resolve for free. Text-join resolution maps built in memory before dependent loads: otByUnivoco (IDUnivoco_IN -> id) for bitacoras.IDOrden_BC, fotos.IDUnivocoOT_FB, repuestos.IDUnivoco_ROT, documentos.IDOrdenes; bitacoraByUnivoco (IDUnivocoBitacora_BC -> id) for fotos.IDBitacora_FB; compraByBusinessKey (IDCompra_C -> id) for detalle.IDCompra_DC and aprobaciones fallback; userByConcatName and userByUsuarioApp for Tecnico_IN/Asignador_OT/Usuario_*/User_*/Tecnico_SS (concat) and UserCarga_IN/UserGen_AP/User_AP/User_SS (usuario_app); edificioByNombre for Edificio_Usr/CentroDeCosto_SS; articuloByCodigo as fallback for the dirty IDArticulo_MS (try Number(value) as id first, else codigo match). Every unresolved join inserts NULL in the FK, preserves the raw value in the *_raw/legacy text column, and appends to an unresolved-joins.csv report - do NOT fail the load, this data is dirty by construction. Type coercion: dates are 'dd/mm/yyyy' strings (parse with strict dd/MM/yyyy, es-AR), hours 'HH:mm', numbers are strings with possible comma decimals (Cantidad_ST, MontoTotal_C) - normalize comma->dot; 'SI'/'NO' -> boolean; empty string -> NULL. Drop all derived Fecha/Mes/Año string columns. Idempotent by design (truncate first); run order enforced by a plain async main, no framework needed.

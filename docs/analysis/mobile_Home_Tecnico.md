# mobile / Home Tecnico

## Purpose
Landing/dispatcher screen for field technicians after login. Shows a personalized greeting, a horizontal carousel of the technician's currently "Asignada" (assigned) work orders, and a role-filtered grid of module tiles (Activos, OT, Ventilaciones, Stock) that either navigate directly or open a building/tower picker before navigating. It performs no writes to SharePoint itself — it is purely a read + collect + navigate hub, plus a logout confirmation flow.

## Layout
Full-screen (640x1136) canvas with a decorative background (001.svg, white rounded card with a blue/gold tools illustration). Top header band: logo (ImgTR_HT, centered ~X198-438,Y44), a transparent hamburger/logout icon-button top-left (bt_cerrarSesios_TC, X18 Y35 60x60... actually W84) and a transparent refresh icon-button top-right (bt_refresh_TC, X545 Y35 60x60). Below that, a large bold greeting "NombreUser" (lbl_nombreTecnico_TC, Y182, size35). Mid-section (Y288-412): horizontal card gallery `gal_home_tecnicos` of assigned OTs (605x124 viewport, 585 template) OR, when empty, a centered "no tasks" illustration (IMG_SinTareas) covering roughly the same band. Lower section (Y516-888): a 2-column tile grid `gal_ModulosTecnico` (589 wide, 372 tall, TemplateSize 292, WrapCount 2) of large rounded-image module buttons (276x171 each). Four full-screen overlays stack on top when active: logout confirm (Group_CST), OT/Ventilaciones tower picker (GroupSeleccionarTorreOT_HT), Stock building picker (GroupSeleccionarTorreST_HT), and a full-screen loading spinner (img_loading_HM) — all implemented as absolutely-positioned image+control groups gated by boolean context variables, i.e. Power Apps' equivalent of modals, not real navigation.

## components
- gal_home_tecnicos (Gallery, horizontal, Items=CollectOT): assigned-OT carousel. Each card = img_gal_HT background (002.svg: white rounded card, blue wrench/tool avatar icon top-left) + Image2_2 priority badge (003/004/005.svg switched by ThisItem.TipoPrioridad_OT, width varies 55/65/61) + lbl_nivel4_OT title (ThisItem.ConcatActivo_IN) + TextCanvas7 subtitle (FechaAsignada_IN & ' | ' & DiasEstimado_IN & ' Días'). Card images call Select(Parent) but the Gallery has no OnSelect override defined in this screen, so tapping a card currently does nothing (no navigation/detail action wired here) — a real gap the React version must decide whether to fix (navigate to OT detail) or intentionally keep as read-only preview.
- IMG_SinTareas (Image, 006.svg): empty-state illustration shown when IsEmpty(CollectOT); purely decorative, no button.
- gal_ModulosTecnico (Gallery, 2-col grid, Items=role-filtered Collect_LPP sorted by field_8): module launcher tiles (img_botones) using ThisItem.Imagen_LPP as the tile image, rounded 8px corners. OnSelect branches on ThisItem.Modulo_LPP to either navigate directly ('Activos') or open a picker modal ('OT','Ventilaciones','Stock').
- bt_cerrarSesios_TC (Button, transparent icon): opens the logout confirmation overlay (Set(CerrarSesion,true)).
- bt_refresh_TC (Button, transparent icon): manually reloads CollectOT with Filter('07.OrdenesTrabajo', Tecnico_IN=NombreUser And Status_OT='Asignada') under a Loading flag.
- lbl_nombreTecnico_TC (Text): greeting, bound to global NombreUser.
- Group_CST logout modal: img_cerrarSesion_1 backdrop+card (007.svg), bt_Cruz_CS_1 (X close), bt_cerrarSesion_1 (confirm, dark filled button), bt_cancelar_CS_1 (cancel, outline button).
- GroupSeleccionarTorreOT_HT tower picker: Img_SelectEdificio_HT backdrop+card (008.svg), bt_cruzSelectTorre_HT (close), cmbox_Torre_HT (ComboBox, Items=Distinct(CollectEdificiosOT,field_2)), bt_cancelarSelectTorre_HT, bt_AceptarSelectTorreOT_HT (disabled until a tower is picked; branches OT vs Ventilaciones flow).
- GroupSeleccionarTorreST_HT stock/building picker: Img_SelectEdificioST_HT backdrop+card (009.svg), bt_cruzSelectTorreST_HT (close), cmbox_TorreST_HT (ComboBox, Items=CollectEdificios, display field Edificio_E via child ComboBoxDataField Edificio_E1), bt_cancelarSelectTorreST_HT, bt_AceptarSelectTorreST_HT (disabled until a building is picked; resolves a hardcoded building→torre/ID group mapping via Switch).
- img_loading_HM (Image, 010.svg, full-screen): loading spinner overlay, Visible=Loading context var toggled around every async block.

## modals
- Logout confirmation (Group_CST / CerrarSesion): full-screen dark backdrop + card, 'Cancelar' (outline) vs 'Cerrar sesión' (filled dark) buttons; opened by the header logout icon, closed by X, Cancelar, or after Cerrar-sesión completes its Navigate(Login).
- Tower picker for OT/Ventilaciones (GroupSeleccionarTorreOT_HT / PopUpSelectTorreOT): backdrop + card with a single ComboBox (Distinct list of towers) and Cancelar/Aceptar; Aceptar disabled until a tower is chosen; routes to either the OT list or the Ventilaciones screen depending on which tile launched it (varEdificio).
- Building picker for Stock (GroupSeleccionarTorreST_HT / PopUpSelectTorreST): backdrop + card with a ComboBox bound to active buildings (CollectEdificios); Aceptar disabled until a building is chosen; resolves a hardcoded building-group/ID mapping before navigating to Screen_Stock.
- Full-screen loading overlay (img_loading_HM / Loading): not interactive, just a blocking spinner shown during every ClearCollect/Refresh block on this screen.

## data_reads
- '07.OrdenesTrabajo' -> CollectOT via bt_refresh_TC: Filter(Tecnico_IN = NombreUser And Status_OT = "Asignada") — drives the home carousel (note: only 'Asignada', unlike the picker flow below).
- '07.OrdenesTrabajo' -> CollectOT via bt_AceptarSelectTorre_HT (OT module flow): Filter(Torre_OT in a hardcoded tower group (or exact match) And (Status_OT="Asignada" Or "Programada" Or "Pendiente")); tower groups: {Palermo Chico, Palermo Soho}, {Palermo Hollywood, Dorrego}, {Montañeses, Nuñez, Hub, Jaramillo}, else exact ThisItem tower.
- '19.Ventilaciones' -> CollectVentilaciones via same accept button (Ventilaciones flow): Filter by field_3/Edificio_VE with the same tower groups And (field_1/Estado_VE = "Asignada" Or "Programada" Or "Pendiente").
- '99.ABM_TipoUnidades' -> CollectEdificiosOT: Filter(Status_ABMUnid = "Alta") — refreshed every time an 'Activos', 'OT' or 'Ventilaciones' tile is tapped; feeds the Distinct(...,field_2) tower combobox.
- '99.ABM_Edificios' -> CollectEdificios: Filter(Status_E = "Activo") — refreshed when the 'Stock' tile is tapped; feeds cmbox_TorreST_HT.
- '08.Stock' -> CollectStock via bt_AceptarSelectTorreST_HT: Filter(FirstN(SortByColumns('08.Stock',"ID",Descending),2000), Text(IDTorreStock) in/= IDEdificio_ST And Status_ST = "Activo") after mapping the selected building to a merged torre label and a ';'-joined ID list (e.g. Hub<->Nuñez share IDs '1;8', Palermo Hollywood<->Dorrego share '4;7', Admin<->Admin 2 share '16;17').
- Collect_LPP (module/permission list, field names Operador_LPP/Tecnico_LPP/Recepcion_LPP/Modulo_LPP/Imagen_LPP/field_8 strongly suggest it is sourced elsewhere, likely App.OnStart, from a list such as 99.ABM_ListaPermisosPerfilesV3 — not populated in this screen's own YAML) -> gal_ModulosTecnico.Items: If PerfilUser="Admin" show all SortByColumns(Collect_LPP,"field_8",Ascending); If "Operador" filter Operador_LPP="SI"; If "Tecnico" filter Tecnico_LPP="SI"; If "Recepecion" (sic) filter Recepcion_LPP="SI". A comment '//Añadir perfil Compras' shows a 5th profile is planned but not implemented.
- PerfilUser and NombreUser: read-only global variables (set by the login flow, not on this screen) used respectively for role gating and the greeting/refresh filter.

## data_writes
- No SharePoint Patch/Remove and no Power Automate flow (.Run) calls anywhere in this screen — it is read/collect/navigate only.
- Only local state mutations: ClearCollect on CollectOT, CollectVentilaciones, CollectEdificiosOT, CollectEdificios, CollectStock (client-side collections, rebuilt each time from the reads above); UpdateContext({Loading:...}) around every async block; Set(varEdificio,...), Set(EdificioIncidente,...), Set(TorreStock,...), Set(IDTorreStock,...) transient routing variables; Set(CerrarSesion,...) and Set(PopUpHamburguesa,false) for the logout overlay; Reset(cmbox_Torre_HT)/Reset(cmbox_TorreST_HT) on cancel/accept.

## navigation
- Navigate('Detalle Activos', Fade) — from the 'Activos' module tile, after ClearCollect(CollectEdificiosOT,...).
- UpdateContext({PopUpSelectTorreOT:true}) — from the 'OT' and 'Ventilaciones' tiles (opens the tower-picker overlay instead of navigating immediately; varEdificio flag distinguishes which flow to run on accept).
- Navigate('Ordenes de Trabajo', Fade) — from the tower-picker Accept button when varEdificio='OT'.
- Navigate(Screen_Ventilaciones, Fade) — from the tower-picker Accept button when varEdificio='VE'.
- UpdateContext({PopUpSelectTorreST:true}) — from the 'Stock' tile (opens the building-picker overlay).
- Navigate(Screen_Stock, Fade) — from the building-picker Accept button.
- Navigate(Login, Fade) + Reset(inputPassword)/Reset(inputUsuario) + reset password-visibility vars — from the logout confirm button (bt_cerrarSesion_1).

## statuses
- Status_OT: "Asignada", "Programada", "Pendiente"
- TipoPrioridad_OT: "Baja", "Media", "Alta"
- Status_ABMUnid (unit/asset status, different domain, coincidentally reuses the word "Alta" to mean 'active/registered' rather than 'high priority'): "Alta"
- Status_E (building status): "Activo"
- Status_ST (stock item status): "Activo"
- Estado_VE / field_1 (ventilation ticket status): "Asignada", "Programada", "Pendiente"
- PerfilUser (role): "Admin", "Operador", "Tecnico", "Recepecion" (typo, see role_logic)
- Modulo_LPP (module key): "Activos", "OT", "Ventilaciones", "Stock"
- Permission flags on Collect_LPP: "SI" (yes/enabled sentinel for Operador_LPP/Tecnico_LPP/Recepcion_LPP)
- varEdificio (internal routing flag): "OT", "VE"

## react_mapping
Header: a mobile TopBar built from two icon Buttons (ghost/ transparent variant) — logout (left) and refresh (right) — plus a plain greeting text bound to the current user's name; not a literal kit primitive, compose from Button+Card slots. Assigned-OT carousel: horizontal-scrolling row of Card components (table-to-cards pattern, since this is a mobile hub, not a dense table) — each Card = leading icon avatar (wrench, reuse 002.svg motif as a small icon badge) + title + subtitle ('date | N días') + a StatusBadge/Badge in the top-right for TipoPrioridad_OT, preserving the existing Baja=blue/Media=yellow/Alta=green mapping unless product decides to correct it; when CollectOT is empty, swap the row for a single empty-state Card (icon + text, no CTA). Decide explicitly whether tapping a card should now navigate to an OT detail view (the legacy screen doesn't wire this) — recommend fixing that gap in the rewrite. Module launcher grid: a 2-column grid of large tappable Cards or icon Buttons (module tile pattern), sourced from a `useModulesForRole(role)` hook/selector (Supabase table + RLS) replacing the Collect_LPP If-chain, so adding a role/module never requires another hardcoded branch. Tower/building pickers: unify into one reusable bottom-sheet Modal (mobile pattern, not a centered dialog) containing a Combobox/Select bound to the relevant dataset (towers vs buildings) plus Cancel/Accept Buttons (Accept disabled via `disabled={!selection}`), parametrized by which module launched it — collapses 008.svg/009.svg's duplicated GroupSeleccionarTorreOT_HT/GroupSeleccionarTorreST_HT into a single component instance. Logout confirmation: standard Modal recipe (confirm dialog) with a secondary Button (Cancelar) and a destructive/primary Button (Cerrar sesión) — center modal is fine here since it's not data-entry. Loading overlay: replace the bespoke Loading-flag full-screen image with the kit's shared loading/Toast infra or a simple full-screen spinner state, not a custom asset. Overall screen composition sits inside the mobile Sidebar/Tab shell used by other technician screens, with this screen as the default/landing tab.
